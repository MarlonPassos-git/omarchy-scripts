# AGENTS.md

## Scope

This repository stores personal scripts used by Omarchy/Hyprland.

## Structure

- Executable scripts live in `src/` and use the `omarchy-` prefix.
- `src/manifest.tsv` is the source of truth for script, title, description, shortcut, and dependencies.
- `scripts/install` installs every command in the manifest.
- `scripts/uninstall` removes every managed keybinding.
- Do not create wrappers in `~/.local/bin` for this project.

## Paths

- Do not commit absolute user paths; use `~` in examples.
- Represent home as `~` in documentation.
- Resolve paths in scripts with `$HOME` or the real clone directory.
- Keybindings must be generated by the installer from `src/manifest.tsv`.

## Hyprland

- Do not edit files in `~/.local/share/omarchy`.
- Use `~/.config/hypr/bindings.conf` for personal keybindings.
- Manage project keybindings inside the `# BEGIN omarchy-scripts` / `# END omarchy-scripts` block.
- After changing keybindings, run `hyprctl reload` and `hyprctl configerrors`.

## Shell

- Scripts must use `set -euo pipefail`.
- Functions should have one responsibility and stay small.
- Prefer `hyprctl ... -j` with `jq` when reading Hyprland state.
- Avoid destructive commands on persistent paths: `rm`, `dd`, `mkfs`, `sudo`, `systemctl`, `shutdown`, `reboot`.
- `rm` is acceptable only for cleaning temporary files created by `mktemp`.
- If a destructive command is unavoidable, ask for explicit confirmation before implementing it.

## Documentation

- Every new script must appear in `README.md`.
- The command table must list title, description, shortcut, script, and dependencies.
- Add code comments only for risks, decisions, and non-obvious side effects.

## Validation

- Run `bash -n <script>` after editing shell scripts.
- Run `./scripts/install` after changing the manifest or installer.
- Run `./scripts/uninstall` after changing uninstall logic.
